SECTION 5: DESIGN DECISIONS
================================================================================

DECISION 1: Remove Phases 7-10 (KV Cache, FP8, Ring Curriculum, Sparse Attn)
  REASON: Experimental, unproven on production hardware
  IMPACT: Clean foundation without research cruft
  PROOF: Saved 400+ lines of broken/incomplete code

DECISION 2: Remove Complex HAL/Allocator System
  REASON: Over-engineered, nobody was using it
  IMPACT: Runtime reduced from 600+ to 400 lines
  PROOF: Three-tier memory was theoretical

DECISION 3: Prefer Recomputation Over Offloading
  REASON: GPUs are compute-heavy, memory is bottleneck
  IMPACT: Simple, safe memory management
  PROOF: torch.checkpoint is battle-tested

DECISION 4: No Custom Kernels in Phase 1
  REASON: Focus on correctness, PyTorch ops are optimized
  IMPACT: Shorter development cycle, less debugging
  PROOF: Latest FSDP and checkpoint APIs are stable

DECISION 5: Separate Runtime, Distributed, Memory, Compiler
  REASON: Single-responsibility principle
  IMPACT: Easy to test, extend, reason about
  PROOF: Each module solves one problem independently

  from zenyx.memory import ActivationManager
  
  scheduler = Scheduler(accumulation_steps=1)
  ActivationManager.hook_into_model(model, use_checkpoint=True)
  
  output = scheduler.forward(model, batch)
  scheduler.backward(loss, optimizer)

EXAMPLE 2: Multi-GPU Training

  model = fsdp.wrap()
  
  output = scheduler.forward(model, batch)
  scheduler.backward(loss, optimizer)
